## Class vs Object
class는 속성에 대한 기술<br>
객체(instance)는 상태 정보를 갖는다.<br>
클래스를 실체화한 것이 객체라고 이해할 수 있다.

## public 선언한 필드는 외부에서 직접 수정할 수 있다.<br>그렇다면 해당 필드에 대한 유효성 검증도 외부에서 수행하는 것이 더 적절할까?
유효성 검증 로직을 외부로 분리하면, 객체를 생성할 때마다 메서드를 호출해야 한다.<br>
번거롭다는 단점도 있지만, 예기치 못한 여러 문제로 인해 잘못된 객체가 생성될 위험성이 존재한다.<br>
새로운 객체가 생성될 때마다 유효성 검사가 필수인 이상 Lion 클래스 생성자에 유효성 검증 로직을 추가하면 안전성과 더불어, 객체가 생성되기 전에 유효성 검사를 할 수 있어 불필요한 객체 생성을 막을 수 있다는 장점도 있다.
